title:CentOS7.4下搭建Elasticsearch6.2.4

准备

Jdk1.8

安装过程

  • 通过java –version确认jdk版本。

    如果出现openjdk请自行百度“Linux openjdk jdk1.8”

  • 登录Elastic官网下载rpm/tar.gz版本。

    sftp传到服务器上，然后tar–zxvf是最安全稳定的方法。在不修改elasticsearch.yml的前提下，elasticsearch也是可以运行的，但是一旦修改注释，让host等语句暴露，普通的服务器主机都会因为参数设置保守的问题被elasticsearch拒绝，这个时候就需要改一些参数，让服务器觉得自己很牛逼。

感谢我的老师姚俊杰博士junjie-yao.info参照腾讯云社区相关指南帮我在root权限下配置好了相关参数。

vm.max_map_count，nofile，nproc三个常见参数配置都在这篇文章中有解答。

  • 选一台无相关的电脑，http://111.111.111.111:9200 登录。

若成功，则说明自用安装存储的搜索引擎基本完成。111.111.111.111为假设的服务器地址。

数据导入

官方API文档：https://www.elastic.co/guide/en/elasticsearch/client/index.html

着重介绍一下批量插入的方法

唯一推荐Bulk API

action_and_meta_data\n
optional_source\n
action_and_meta_data\n
optional_source\n
....
action_and_meta_data\n
optional_source\n

原因主要是json格式的普及和RESTful风格的便利。

多线程并不能提高插入效率,参见有位CSDN老哥的实验

Bulk API 的使用总体注意点有三点：

  • 第一行
{"index":{"_index":"insuranc","_type":"health12","_id": 3 }}

_index, _type, _id缺一不可，注意花括号。

  • 第二行
{"company":"太平人寿保险有限公司","product":"太平个人税收优惠型健康保险B款（万能型）","type1":"健康保险"}

通用json格式，特别注意一点是右花括号左边和右边都不能有逗号，在使用csv2json等插件转译过程中会出现这样的情况。

  • 严重注意‘\n’

观察官方文档格式，每组数据的第二行都有回车，这意味着所有肉眼可见的正文结束后，文件尾需要有回车换行。

依照Bulk API要求格式，对于json文件精加工，具体参见代码。

curl -H "Content-Type: application/json" -s -XPOST localhost:9200/_bulk?pretty --data-bin
ary @gotobed.json

?pretty是美化输出，“errors” :false说明操作成功。

数据导入验证

1.http://111.111.111.111:9200/_cat/indices?v

可以查看所有索引情况。

2.http://111.111.111.111:9200/刚才插入的_index的值/刚才插入的_type的值/一个存在的_id

可以准确验证单条数据的导入准确度。

3.http://111.111.111.111:9200/insur/heal/_search?q=string

string可以是company:幸福人寿，也可以是company:幸福人寿&size=100
